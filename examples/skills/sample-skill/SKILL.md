# Sample Skill

## Workflow

1. Read the user request.
2. Create or update a small output file.
3. Explain what changed.
4. Verify success by checking that the output file exists.

## Boundaries

Only handle simple file-writing tasks. Do not claim broad product, security, or platform support.

## Success

The expected result is a concrete file on disk and a final answer explaining the change.
